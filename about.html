What This Website Is

This website is a play-helper for building Here to Slay DUNGEONS layouts quickly on a digital board. It handles tile selection, drag-and-drop placement, snapping to the board grid, tile rotation, and boss-card handling in one place.

The core goal is speed: choose a tile set, get your active dungeon tiles, place them with visual feedback, and finalize with a boss. It is not a replacement for your full game setup, but it is meant to remove setup friction and reduce rule-checking overhead while building your map.

The board is interactive and designed for iteration. You can pan the board, zoom in for precise placement, reroll selected tiles, and reset when needed. The interface keeps tiles and boss options organized in drawers so you can focus on layout decisions instead of file juggling.

Manual

This manual is intentionally detailed and long-form. It is meant to be scrolled and used as reference while you work. If you are brand new, start with the Quick Start flow below, then read each feature section.

Quick Start Flow

  1. Choose a Tile Set from the top bar.
  2. Use the six selected tiles in the Tile Drawer as your active build tiles.
  3. Drag tiles onto the board and rotate as needed.
  4. Use placement feedback and status text to confirm valid placement behavior.
  5. Select a boss from the Boss Selection area and drag it to the board.

Feature: Tile Set Selector

The Tile Set selector changes which dungeon assets are loaded for the current round. When you switch sets, the app updates entrance tile art, regular tile art, reference card art, and boss art for that set. If a set is not fully ready, it can appear as unavailable in the selector.

Practical use: pick your intended environment first (for example Molten or Overgrown), then continue setup from there. Switching tile sets mid-planning is possible, but it resets flow and should usually be done before serious placement work starts.

Feature: Selected Tiles (Tile Drawer)

The left drawer contains the six active dungeon tiles for the current round. These are the tiles you can place on the board to form the run layout. Drag from the drawer to begin placing, or return tiles to the drawer by dropping them back into the drawer area.

Tiles in this area remain easy to compare at a glance. The drawer is built for quick pick-and-place loops: pick a tile, test a location, rotate, drop, evaluate feedback, and keep moving.

Feature: Reroll Tiles

The Reroll Tiles control in Selected Tiles replaces only tray selection for regular dungeon tiles. It is useful when the current active six do not produce the shape variety you want. Existing board context and interface state continue so you can iterate quickly.

Use reroll early when you recognize a dead-end tile combination. It is faster than manually forcing a difficult arrangement that does not fit your intended dungeon pathing.

Feature: Reserve Tiles

Tiles not currently active stay in Reserve Tiles. Reserve exists so all set tiles remain visible in-session without cluttering the main build surface. Think of it as parking space for non-active options.

Reserve is mainly informational during normal flow and helps you understand what is excluded from the current active six. If you are optimizing for different route shapes, reserve visibility helps you decide whether to reroll.

Feature: Board Interaction (Drag, Drop, Pan, Zoom)

The board is the main workspace where tiles and boss tokens are placed. Drag tiles onto the grid and release to place. While dragging in board space, snapping and magnet behavior keep placement aligned to the lattice.

Pan is available by dragging empty board space. Zoom is available with mouse wheel over the board. These two controls matter when your map grows beyond the initial center framing.

Use zoom-in for tight adjacency checks, then zoom-out to validate overall shape and traversal flow. Alternating between local precision and global overview is the fastest way to avoid late layout corrections.

Feature: Rotation Controls

Tiles can be rotated in 60-degree increments. Rotation can be triggered from keyboard and on-tile rotation controls. Rotation is essential for matching faces and improving local fit.

Recommended workflow: move tile near target, rotate until contact geometry looks plausible, then drop and validate. This minimizes failed placements and reduces repeated long-distance dragging.

Feature: Placement Feedback & Status

The right drawer provides feedback indicators and status text while you build. It helps confirm whether your current board state is progressing toward a valid complete setup. Use this area as your immediate readout for success/failure context after every drop.

If a placement is not valid, feedback communicates that immediately so you can adjust without guesswork. The status area is also useful for understanding state changes after key actions like reset and random placement.

Feature: Boss Selection

Boss Selection in the right drawer lets you cycle boss cards and place one as a board token. This keeps encounter choice integrated with map planning instead of being a disconnected later step.

The random boss action is useful when you want unbiased boss selection. Boss token placement uses board-relative behavior so it can stay coherent while navigating the workspace.

Feature: Auto Build Dungeon

Auto Build generates a complete board arrangement from active selected tiles using the app's placement logic. It is best used when you want a fast starting point or when manually exploring many alternatives would take too long.

Treat auto-build output as a candidate layout, not a final mandate. You can still move, rotate, and fine-tune placements afterward to match preferred flow or visual balance.

Feature: Reset Tiles And Boss Cards

Reset returns the session to a clean board state for the current tile set. It is the fastest way to restart if the current layout direction no longer makes sense. Reset is also useful after testing several auto-build runs and wanting a clear baseline.

Feature: Save Layout / Load Layout

Save Layout exports your current arrangement to file. Load Layout restores a saved arrangement. This is intended for continuity across sessions and for comparing multiple layout ideas without losing progress.

Recommended practice: save after major milestones (first stable build, post-rotation pass, final boss placement). Keeping snapshots makes experimentation safer because rollback is trivial.

Feature: Drawer Controls

Both side drawers can be collapsed and expanded. This gives more visual space when focusing on board geometry and less on controls. Collapsing drawers is especially useful on smaller screens or during fine drag operations.

Shortcuts

Keyboard and pointer controls currently available:

  • W: Rotate hovered/selected tile counter-clockwise (-60 degrees).
  • E: Rotate hovered/selected tile clockwise (+60 degrees).
  • R: Run random placement (Auto Build Dungeon).
  • X: Reset tiles and boss cards.
  • D: Toggle both drawers collapsed/expanded.
  • B: Random boss selection/placement action.
  • Escape: Close open top-menu dropdowns.
  • Mouse wheel over board: Zoom board in/out.
  • Drag empty board space: Pan board.
  • Drag tile or boss token: Move item in workspace.

Shortcut behavior is context-sensitive to avoid accidental triggers while typing in inputs. If a shortcut does not fire, click back on the board area first and try again.

Practical Build Tips

Start by placing only two or three tiles around the entrance and evaluate directional flow before committing to full expansion. Early structure decisions create most downstream constraints, so this initial pass is high leverage.

Keep one cycle of "place, rotate, zoom-check, step back" as your default rhythm. Rapid alternating between detail and overview catches misalignment early and reduces major rebuilds.

If you hit repeated invalid outcomes, reroll active tiles sooner rather than later. Spending too long forcing an incompatible tile set usually costs more time than a controlled reset-and-retry cycle.
